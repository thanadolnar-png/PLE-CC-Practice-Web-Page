## เว็บนี้ทำอะไรได้บ้าง?

## ส่วนที่ 1 — คลังเคส (ดูเคสได้อิสระ)

ดูเคสสอบ OSPE ทั้งหมดที่ทีมเตรียมไว้ โดยไม่ต้องเปิด Google Docs เอง
เนื้อหาดึงมาจาก Google Docs โดยตรง อัปเดตอัตโนมัติเมื่อทีมแก้ไข

สิ่งที่ทำได้ในส่วนนี้:

1. กรองเคสตามหมวด
   - เลือกดูแค่ Clinic / Product / SAP อย่างใดอย่างหนึ่ง หรือดูทั้งหมด

2. กรองตาม OSPE Main Group หรือ Station/Sub-topic
   - เช่น อยากดูเฉพาะกลุ่ม "การให้คำแนะนำยา" หรือสถานี "Warfarin Counseling" ก็เลือกได้เลย

3. ค้นหา
   - พิมพ์ชื่อโรค ชื่อยา หรือคำสำคัญ แล้วเว็บจะหาเคสที่ตรงให้

4. กดเปิดเคสแล้วอ่านเนื้อหาได้เลย
   - เห็นโจทย์, ข้อมูลผู้ป่วย, รูปภาพ (ถ้ามี) ทุกอย่างเหมือนใน Google Docs
   - มี Checklist ให้กดติ๊กได้เป็นข้อๆ พร้อมคะแนนแต่ละข้อ
   - เห็นคะแนนสะสมแบบ real-time ขณะติ๊ก
   - ระบบจำว่าติ๊กไปถึงไหนแล้ว ปิดแล้วเปิดใหม่ก็ยังอยู่

---

## ส่วนที่ 2 — ซ้อมสอบจริง (เหมือนวันสอบ)

ซ้อมแบบ OSPE จริงๆ มีจับเวลา มีข้ามสถานีได้ เหมือนนั่งสอบ

ขั้นตอนก่อนเริ่มซ้อม:

1. เลือกจำนวนสถานี
   - ปกติ 16 สถานี หรือจะซ้อมแค่ 5-10 สถานีก็ได้

2. กำหนดสัดส่วน
   - เช่น อยากโฟกัส Clinic เยอะหน่อย ก็ปรับ Clinic 10 Product 4 SAP 2
   - หรือปล่อยให้ระบบใช้สัดส่วนมาตรฐาน (Clinic 8 : Product 6 : SAP 2)

3. เลือก Scope (ไม่บังคับ)
   - กรองตาม OSPE Main Group เช่น เลือกเฉพาะกลุ่ม "การเตรียมยา" หรือ "การให้คำแนะนำยา"
   - กรองตาม Station/Sub-topic เช่น "Warfarin" หรือ "Cream Compounding" เท่านั้น
   - หรือกดสุ่มทั้งหมดเลย

4. กดเริ่ม — ระบบจะสุ่มเคสมาให้ครบตามที่เลือก

ระหว่างซ้อม:

- เห็นเนื้อหาเคสเต็มจอ พร้อม Checklist + คะแนนด้านขวา
- มีนาฬิกานับถอยหลัง 4 นาที
  - สีเขียว = ยังมีเวลาเหลือ
  - สีเหลือง = เหลือน้อยกว่า 1 นาที
  - สีแดง = ใกล้หมดเวลา
- ปุ่มหยุด / เริ่มใหม่ นาฬิกา (ถ้าต้องการหยุดพัก)
- ปุ่มสถานีถัดไป และ ย้อนกลับ
- แผนที่สถานี 16 ช่อง บอกว่าอยู่สถานีไหน ทำไปแล้วกี่ข้อ

หลังซ้อมเสร็จ:

- ดูสรุปผลคะแนนแยกตาม OSPE Main Group ว่าได้กี่ % ในแต่ละกลุ่มทักษะ
- ดูเวลาเฉลี่ยต่อสถานี
- พิมพ์หรือบันทึกผลได้

## ระบบ Tag 2 ระดับ + คะแนน Checklist

นี่คือหัวใจสำคัญของระบบใหม่

---

### Tag ระดับที่ 1 — OSPE Main Group (กลุ่มทักษะหลัก)

คือกลุ่มทักษะใหญ่ๆ ที่ครอบคลุมทั้งหมวด Clinic / Product / SAP
ใช้สำหรับกรองและดูสรุปคะแนนในภาพรวม

ตัวอย่าง OSPE Main Group:

หมวด Clinic:
  - การซักประวัติและประเมินผู้ป่วย
  - การให้คำแนะนำยา (Pharmacy Counseling)
  - การแก้ปัญหาการใช้ยา (Drug Therapy Problem)
  - การบริการข้อมูลยา (Drug Information)
  - การติดตามการรักษา (Monitoring & Follow-up)

หมวด Product:
  - การคำนวณตำรับและขนาดยา
  - การเตรียมยา (Compounding)
  - การเขียนฉลากและจ่ายยา
  - การควบคุมและประกันคุณภาพ (QA/QC)

หมวด SAP:
  - ความรู้กฎหมายและจรรยาบรรณ
  - การจัดการข้อมูลสุขภาพ
  - เศรษฐศาสตร์และนโยบายสุขภาพ

---

### Tag ระดับที่ 2 — Station / Sub-topic (หัวข้อทักษะย่อย)

คือสถานีหรือหัวข้อเฉพาะ บอกว่าเคสนี้เกี่ยวกับอะไรโดยตรง
ใช้สำหรับกรองเคสแบบละเอียด และติดตามว่าตัวเองอ่อนเรื่องอะไร

ตัวอย่าง Station/Sub-topic:

  Main Group: การให้คำแนะนำยา
    -> Warfarin Counseling
    -> Insulin Injection Technique
    -> Inhaler Device Counseling
    -> OCP Counseling

  Main Group: การเตรียมยา
    -> Cream Compounding
    -> Oral Liquid Compounding
    -> TPN Preparation
    -> Sterile Reconstitution

### ระบบคะแนน Checklist

แต่ละข้อใน Checklist มีคะแนนของตัวเอง กำหนดโดยทีมผู้เขียนเคส

การแสดงผลบนเว็บ:

  [ ] ทักทายผู้ป่วยและแนะนำตัว                  (1 คะแนน)
  [ ] ยืนยันชื่อและวันเกิดผู้ป่วย                (1 คะแนน)
  [ ] อธิบายชื่อยาและกลไกการออกฤทธิ์            (2 คะแนน)
  [ ] อธิบายวิธีใช้ยาและขนาดยาที่ถูกต้อง        (2 คะแนน)
  [ ] แจ้งผลข้างเคียงที่สำคัญ                   (3 คะแนน)
  [ ] แนะนำการสังเกตอาการและเมื่อไรควรพบแพทย์   (2 คะแนน)
  --------------------------------------------------
  คะแนนที่ได้: 0 / 11 คะแนน  (เกณฑ์ผ่าน: 80% = 8.8 คะแนน)

เมื่อกดติ๊ก:
  - คะแนนสะสมเพิ่มขึ้นทันที
  - แถบความคืบหน้าแสดง % ที่ได้
  - ถ้าถึง 80% แถบเปลี่ยนเป็นสีเขียว (ผ่านเกณฑ์)
  - ถ้ายังต่ำกว่า 80% แถบเป็นสีแดง (ยังไม่ผ่าน)

---

### โครงสร้างข้อมูล Checklist ข้อเดียว (ที่ระบบเก็บไว้)

แต่ละข้อใน Checklist เก็บข้อมูลดังนี้:

  - ข้อความ: สิ่งที่ต้องทำหรือพูด
  - คะแนน: น้ำหนักคะแนนข้อนี้ (เช่น 1, 2, 3)
  - Main Group: กลุ่มทักษะหลักที่ข้อนี้จัดอยู่
  - Sub-topic: หัวข้อย่อยที่เฉพาะเจาะจงกว่า
  - ติ๊กแล้วหรือยัง: true / false

---

### ดูสรุปคะแนนได้แยกตาม Main Group

หลังซ้อมเสร็จ เว็บจะแสดงสรุปแบบนี้:

  ผลการซ้อม — สถานีที่ 3: Hypertension Counseling
  -------------------------------------------------------
  กลุ่มทักษะ                       | ได้   | เต็ม | %
  ----------------------------------|-------|------|-----
  การให้คำแนะนำยา                  |   7   |   8  | 87%  ✓
  การแก้ปัญหาการใช้ยา              |   2   |   3  | 67%  ✗
  -------------------------------------------------------
  รวม                               |   9   |  11  | 82%  ✓ ผ่าน

เกณฑ์ผ่าน OSPE = 80% ของคะแนนทั้งหมดต่อสถานี

```
# Warfarin Counseling — ผู้ป่วย AF ใหม่

## ข้อมูลเคส
- หมวด: Clinic
- OSPE Main Group: การให้คำแนะนำยา
- Station/Sub-topic: Warfarin Counseling
- โรค/หัวข้อ: Atrial Fibrillation, Anticoagulation
- ระดับ: 3
- ผู้เขียน: Lin
- วันที่: 15/06/2569

## โจทย์
ผู้ป่วยชาย อายุ 62 ปี วินิจฉัย AF ใหม่ แพทย์สั่ง Warfarin 3 mg OD
คุณเป็นเภสัชกรที่ต้องให้คำแนะนำยาก่อนผู้ป่วยกลับบ้าน

## ข้อมูลผู้ป่วย
| รายการ     | ข้อมูล              |
|------------|---------------------|
| ชื่อ       | นายสมชาย ใจดี       |
| อายุ       | 62 ปี               |
| โรคประจำตัว | AF, HT, DM         |
| ยาปัจจุบัน | Warfarin 3mg, Metformin 500mg |
| ค่า INR ล่าสุด | 1.2 (วันนี้)   |

## Checklist [คะแนนเต็ม: 14 คะแนน]

### (กลุ่ม: การให้คำแนะนำยา)
[ ] (1) ทักทายและแนะนำตัวเองในฐานะเภสัชกร
[ ] (1) ยืนยันชื่อ-สกุลและ HN ผู้ป่วย
[ ] (2) อธิบายว่า Warfarin คืออะไรและทำไมต้องใช้
[ ] (2) อธิบายวิธีกิน ขนาดยา และเวลาที่ควรกิน
[ ] (2) แนะนำการตรวจ INR สม่ำเสมอ และบอกค่า Target INR (2-3)

### (กลุ่ม: การแก้ปัญหาการใช้ยา)
[ ] (2) อธิบายอาการเลือดออกผิดปกติที่ต้องระวัง
[ ] (2) แนะนำอาหารที่ควรหลีกเลี่ยงหรือทานสม่ำเสมอ (Vitamin K)
[ ] (2) บอกยาที่ห้ามซื้อกินเองโดยไม่ปรึกษาแพทย์/เภสัชกร (NSAID, ASA)

## หมายเหตุ / เฉลย
- Target INR สำหรับ AF = 2.0-3.0
- อาหาร Vit K สูง: ผักใบเขียว ตับ น้ำมันพืชบางชนิด
- ยาที่มี interaction สำคัญ: Aspirin, NSAIDs, Amiodarone, Fluconazole
```

### สิ่งที่จะเกิดขึ้นถ้า Tag ไม่ตรงกับ Sheet

| ปัญหา | ผลที่เกิด |
|--------|----------|
| Course Group สะกดผิด | เคสจะไม่ขึ้นเมื่อกรอง Course Group นั้น |
| Main Group ไม่มีใน Sheet | กราฟสรุปคะแนนจะไม่มีกลุ่มนั้น |
| Sub-topic ไม่ตรง | ค้นหาเคสด้วยชื่อสถานีไม่เจอ |
| หมวด (Clinic/Product/SAP) ผิด | เคสหายไปจากหมวดที่ถูกต้อง |
